# Цифровой профпатолог — MVP

Демонстрационный прототип рабочего места врача-профпатолога.

## Что умеет

- загружает CSV с осмотрами;
- ищет пациента по `patient_id`;
- показывает все осмотры пациента;
- раскрывает выбранный `exam_row_id`;
- показывает факторы вредности, заключения специалистов, исследования/анализы и рекомендации;
- запускает ML-заглушку по кнопке «Предсказать»;
- подсвечивает связи: при наведении на красный фактор подсвечиваются заключения и исследования, которые повлияли на вывод;
- формирует строку ответа в формате хакатона: `exam_row_id,factors`;
- позволяет скачать CSV-ответ.

## Запуск

```bash
cd digital_propathologist_mvp
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
streamlit run app.py
```

После запуска откройте адрес, который Streamlit покажет в терминале.

## Данные

Можно загрузить исходный `train2.csv`. Также в проекте есть небольшой синтетический `data/demo_sample.csv`, чтобы быстро проверить интерфейс без большого файла.

Ожидаемые колонки:

- `exam_row_id`
- `patient_id`
- `consultation_date`
- `assigned_harmful_factors`
- `specialist_conclusions`
- `contraindicated_factors`
- `has_contraindications`

## Где подключать реальную ML-модель

Файл:

```text
digital_propathologist/prediction_stub.py
```

Функция:

```python
predict_exam(exam: Exam) -> PredictionResult
```

Сейчас она использует безопасную демонстрационную заглушку. Ее можно заменить на:

- импорт вашей Python-модели;
- вызов FastAPI/Flask API;
- загрузку `.pkl`, `.joblib`, `.pt` или другого артефакта модели.

## Важное ограничение

Прототип не выносит медицинское заключение. Формулировки интерфейса намеренно используют слова: «предварительный вывод», «требуется внимание профпатолога», «черновик для врача». Финальное решение принимает врач.
